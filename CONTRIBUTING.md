# Contributing

## Quick Flow

1. Create a branch from `main`.
2. Make focused changes.
3. Verify behavior locally.
4. Open a pull request on GitHub with a clear summary.

## Scope Guidelines

- Keep changes small and reviewable.
- Avoid unrelated refactors in the same PR.
- Keep all user-facing text in English.
- Update docs when behavior or setup changes.

## Feature Work

For new feature development, follow:
[`docs/add-feature.md`](docs/add-feature.md)

## Pull Request Notes

Include:

- What changed
- Why it changed
- How it was validated
- Any known limitations

## Code Style

- Follow existing Kotlin and XML patterns in the repo.
- Prefer clear naming over clever abstractions.
- Keep feature logic isolated to its feature package.
